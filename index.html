🧠 What is MCP?
Understanding the concept before we install anything. Read this first — it makes everything else make sense.
MCP = Model Context Protocol. It is an open standard created by Anthropic (the company that makes Claude).

The problem it solves: By default, Claude is like a very smart person locked in a room with no internet, no files, no access to the outside world. It can only read and respond to what you type.

What MCP does: MCP servers are like doors out of that room. Each server gives Claude access to one thing — your files, the internet, a database, job boards, a browser, etc.

Think of it like phone apps: Without apps, a phone can only make calls. With apps, it can do anything. Without MCP, Claude can only chat. With MCP servers, Claude can search for jobs, read your resume, fill application forms, and more.

4
Create the job search server file
💡 How to create the file on Windows
Open Notepad → paste the code below → File → Save As → navigate to C:\Users\YourName\job-automation\ → filename: job_search_server.py → Save as type: All Files → Save.
python — job_search_server.py
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import Tool, TextContent
import json
from jobspy import scrape_jobs

app = Server("job-search")

# This function tells Claude what tools are available
@app.list_tools()
async def list_tools():
    return [
        Tool(
            name="search_jobs",
            description="Search for jobs across LinkedIn, Indeed, Glassdoor, Naukri",
            inputSchema={
                "type": "object",
                "properties": {
                    "search_term": {"type": "string", "description": "Job title to search"},
                    "location": {"type": "string", "description": "City or country"},
                    "results_wanted": {"type": "integer", "default": 10},
                    "site_name": {
                        "type": "array",
                        "items": {"type": "string"},
                        "default": ["indeed", "linkedin"]
                    },
                    "is_remote": {"type": "boolean", "default": False}
                },
                "required": ["search_term", "location"]
            }
        )
    ]

# This function runs when Claude calls search_jobs
@app.call_tool()
async def call_tool(name: str, arguments: dict):
    if name == "search_jobs":
        jobs = scrape_jobs(
            site_name=arguments.get("site_name", ["indeed", "linkedin"]),
            search_term=arguments["search_term"],
            location=arguments["location"],
            results_wanted=arguments.get("results_wanted", 10),
            is_remote=arguments.get("is_remote", False),
        )
        results = []
        for _, row in jobs.iterrows():
            results.append({
                "title": str(row.get("title", "")),
                "company": str(row.get("company", "")),
                "location": str(row.get("location", "")),
                "job_url": str(row.get("job_url", "")),
                "description": str(row.get("description", ""))[:500],
                "date_posted": str(row.get("date_posted", "")),
            })
        return [TextContent(type="text", text=json.dumps(results, indent=2))]

async def main():
    async with stdio_server() as (read, write):
        await app.run(read, write, app.create_initialization_options())

if __name__ == "__main__":
    import asyncio
    asyncio.run(main())

4
Replace all content with this config — update YOUR_USERNAME in 3 places
❌ Important — Replace YOUR_USERNAME
The config below has YOUR_USERNAME in 3 places. Replace ALL 3 with your actual Windows username from Step 1. Also use double backslashes \\ in paths on Windows.
json — claude_desktop_config.json
{
  "mcpServers": {
    "job-search": {
      "command": "C:\\Users\\YOUR_USERNAME\\job-automation\\.venv\\Scripts\\python.exe",
      "args": [
        "C:\\Users\\YOUR_USERNAME\\job-automation\\job_search_server.py"
      ]
    },
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "C:\\Users\\YOUR_USERNAME\\job-automation"
      ]
    },
    "duckduckgo": {
      "command": "npx",
      "args": ["-y", "duckduckgo-mcp-server"]
    }
  }
}

🚑 Fix Errors
Find your error below and follow the exact fix steps.
Error: 'uv' is not recognized as an internal or external command
Cause: uv was installed but Windows PATH wasn't updated
fix — cmd
# Close ALL Command Prompt windows
# Open a fresh one as Administrator
# Then try again:
uv --version

# If still not found, add to PATH manually:
# Windows key → "Environment Variables" → System Variables
# → Path → Edit → New → Add: C:\Users\YourName\.local\bin
# → OK → OK → restart CMD
Error: No module named 'jobspy' or No module named 'mcp'
Cause: venv is not activated or packages not installed
fix — cmd
cd %USERPROFILE%\job-automation
.venv\Scripts\activate
# Make sure you see (job-automation) in prompt
uv pip install python-jobspy mcp
# Verify:
python -c "import jobspy; print('jobspy OK')"
python -c "import mcp; print('mcp OK')"
Error: Hammer icon / tools not showing in Claude Desktop
Cause: Claude was not fully restarted after config change
fix — steps
# Fully quit Claude Desktop:
# Right-click Claude icon in taskbar → Quit
# OR: Task Manager → find Claude → End Task
# Wait 5 seconds
# Reopen from Start Menu
# Wait 10 seconds — servers take time to start
# Click the + button at bottom of chat input
Error: The system cannot find the path specified
Cause: Python path in config.json is wrong
fix — cmd
# Get the correct python path:
cd %USERPROFILE%\job-automation
.venv\Scripts\activate
where python
# Copy the output (e.g. C:\Users\student01\job-automation\.venv\Scripts\python.exe)
# Open config file and paste this as the "command" value
# Remember to double every backslash: C:\\Users\\...

notepad %APPDATA%\Claude\claude_desktop_config.json
Error: npx is not recognized
Cause: Node.js is not installed or not in PATH
fix — cmd
node --version
npm --version
# If either fails → reinstall Node.js from https://nodejs.org
# Use LTS version, run installer with default options
# Close and reopen CMD after install
Error: Server connected but returns no jobs
Cause: Job boards may be rate-limiting or temporarily blocking
fix — try different sites
# In Claude Desktop, try a different site_name:
# "Use search_jobs with site_name: ['indeed'] to find React jobs in Chennai"
# Or:
# "Use search_jobs with site_name: ['glassdoor'] to find Frontend jobs"
# Wait 5 minutes between attempts to avoid rate limiting
